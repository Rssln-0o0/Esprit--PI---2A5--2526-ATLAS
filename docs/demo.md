# Demonstration

## Objectif

La demonstration permet de verifier que l'application desktop, la base de donnees et les composants IoT fonctionnent ensemble. Elle est indispensable pour un projet embarque ou IoT, car le materiel ne peut pas toujours etre reproduit immediatement par un evaluateur externe.

## Video de demonstration

Lien actuel :

```text
https://www.youtube.com/watch?v=n4aDg8cYl-A
```

Le lien est egalement disponible dans :

```text
demo/Video Démo Link.txt
```

## Contenu recommande de la video

La video doit montrer :

- le lancement de l'application Qt ;
- une connexion reussie a la base de donnees ;
- au moins une operation CRUD principale ;
- la communication avec une carte Arduino ou ESP32 ;
- la reaction d'un capteur ou d'un actionneur ;
- le fonctionnement d'un module specifique comme RFID, GPS, ESP32-CAM, stock ou assistant.

## Captures d'ecran recommandees

Pour completer la video, ajouter des captures dans :

```text
demo/screenshots/
```

Captures conseillees :

- ecran principal ;
- module Employes ;
- module Stock ;
- module Quai ou Bateau ;
- interaction IoT ou retour capteur ;
- exemple de QR code, detection visage ou assistant stock.

## Verification avant publication

Avant la soumission finale, verifier que :

- le lien video est public ou accessible aux evaluateurs ;
- la video ne montre aucun secret, mot de passe ou cle API ;
- les donnees affichees sont fictives ou anonymisees ;
- les fichiers de demonstration ne depassent pas les limites GitHub ;
- les captures et videos correspondent bien a la version presente dans le depot.
